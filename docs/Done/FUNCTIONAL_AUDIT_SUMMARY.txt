ARCHITECTURE OBSERVATIONS
================================================================================

PATTERN: Hollow Component Architecture (All Pages)
- Page Component: ONLY JSX/rendering logic
- Logic Hook (use*PageLogic.js): ALL business logic, state, effects
- Sub-components: Modular UI components for sections
- CSS: Component-scoped styles

DATA FLOW:
1. User Action → Component event handler
2. Logic Hook → Processes action, manages state
3. Supabase Call → Edge Function → Database/API
4. Response → Hook updates state
5. Re-render → UI displays new data

REUSABLE COMPONENTS:
- AdminHeader (all pages)
- FilterBar (most pages)
- Modal dialogs (various types)
- State components (Loading, Error, Empty)
- Toast notifications
